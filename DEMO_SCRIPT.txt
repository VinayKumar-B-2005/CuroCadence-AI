[0:20 WHAT IT IS]

"CuroCadence AI is a personal medication concierge — 
built on the Google Agent Development Kit's Concierge Agents track.

It's not a calendar. It's not a pill reminder app.
It's an intelligent multi-agent AI system that:
  — Manages your full medication schedule
  — Catches dangerous drug interactions before you take a pill
  — Pauses and asks a human to review when the risk is too high to proceed automatically
  — Escalates to your caregiver or pharmacist the moment something looks wrong
  — And keeps a secure, auditable trail of every single decision it makes

The tagline: Safe. Smart. Always Watching Your Meds."

[1:00 ARCHITECTURE]
(show assets/architecture_diagram.png)

"Every message a user sends enters the pipeline here — 
at the Security Checkpoint.

Before ANY text reaches an AI agent, we run two checks:
  One: PII scrubbing — phone numbers, addresses, names — 
       redacted from all logs before they're written.
  Two: Prompt injection detection — 
       if someone types 'ignore prior instructions' or 'jailbreak',
       we catch it, block it, log it as CRITICAL, and the agent 
       never sees that input. Test case three will show this live.

If the input is safe, it reaches the Orchestrator.

The Orchestrator is the root LlmAgent — it reads the intent 
and routes to the right specialist:

  The Scheduler Agent manages your medication list. 
  It understands natural language — 'take metformin twice daily after meals' 
  gets parsed into structured schedule entries automatically.
  It uses four MCP tools: add_medication, get_medication_schedule, 
  log_dose_taken, and export_schedule_ics.

  The Interaction and Safety Agent checks every new drug 
  against your existing medications using a local lookup table — 
  no external medical API needed, data stays private. 
  It flags severity LOW, MEDIUM, or HIGH.

  When the severity is MEDIUM or HIGH — we don't just show a warning 
  and move on. We PAUSE. We invoke what ADK calls RequestInput — 
  a human-in-the-loop approval step. No one gets an interaction 
  that's flagged HIGH without a real human clicking Approve.

  If it's HIGH, the Caregiver Liaison Agent drafts a professional alert 
  with the interaction details, the risk description, and recommended action.
  For true emergencies, there's a CRITICAL escalation path 
  that bypasses the normal flow entirely.

  And the Reporting Agent tracks your weekly adherence — 
  doses taken versus scheduled, streak count, missed doses.

On the right: the MCP Server — six tools exposed via the Model Context Protocol —
used by multiple agents throughout the pipeline."

(pause)

[3:00 SECURITY & MCP]

"Two quick highlights:

Security: three layers — PII scrubbing, injection detection, 
and a hard-coded domain rule that HIGH-severity interactions 
can NEVER be auto-approved. That's enforced in code, not by the LLM.

MCP: six tools — get_medication_schedule, add_medication, 
check_interaction, log_dose_taken, export_schedule_ics, 
get_adherence_report — all exposed via the Model Context Protocol 
and wired into the Scheduler and Interaction agents.
In production, these would connect to real pharmacy databases 
or send real SMS alerts through Twilio."
